# Redux Page Module Generator Agent

## Purpose
Generate new page modules with Redux integration following the established patterns in the luyenthi-client project.

## Code Patterns Reference

### Redux Store Structure
- **Store**: Uses Redux Toolkit with configureStore
- **Location**: `src/redux/store.ts`
- **Slices Location**: `src/redux/slice/`
- **Hooks**: Custom typed hooks in `src/redux/hooks.ts`

### Slice Pattern (Based on userSlice.ts)
```typescript
// 1. Define async thunks first
export const fetchData = createAsyncThunk(
  "moduleName/fetchData",
  async (params: IParams) => {
    const res = await callFetchData(params);
    return res.data;
  }
);

// 2. Define state interface
interface IState {
  isFetching: boolean;
  data: IDataType[];
  meta?: {
    limit: number;
    page: number;
    total: number;
  };
}

// 3. Create slice with reducers + extraReducers
const slice = createSlice({
  name: "moduleName",
  initialState,
  reducers: { /* synchronous actions */ },
  extraReducers: (builder) => {
    builder
      .addCase(fetchData.pending, (state) => { state.isFetching = true; })
      .addCase(fetchData.fulfilled, (state, action) => { /* handle success */ })
      .addCase(fetchData.rejected, (state) => { state.isFetching = false; });
  }
});
```

### API Function Pattern
- **Location**: `src/config/api.ts`
- **Pattern**: Named exports like `callFetchUser`, `callCreateUser`, `callUpdateUser`, `callDeleteUser`
- **Returns**: Axios promise that resolves to response

### Page Component Pattern
- **Location**: `src/pages/[ModuleName]/`
- **Structure**: Functional components using hooks
- **Redux Integration**: Uses `useAppDispatch` and `useAppSelector` from redux hooks
- **Forms**: Uses `react-hook-form` with `@hookform/resolvers`
- **UI**: Uses shadcn/ui components via Tailwind

### Type Definitions
- **Location**: `src/types/backend.ts`
- **Pattern**: TypeScript interfaces for API responses and data models
- **Common Types**:
  ```typescript
  interface IBackendRes<T> {
    success: boolean;
    data?: T;
    message?: string;
    meta?: IMeta;
  }
  
  interface IMeta {
    limit: number;
    page: number;
    total: number;
  }
  ```

## Generation Requirements

When asked to create a page module, follow these steps:

### 1. Gather Information
Ask for:
- **Module/Resource Name** (e.g., "Product", "Category", "Order")
- **API Endpoints** (list endpoints: GET list, GET single, POST create, PUT update, DELETE)
- **Main Fields** (object properties with types)
- **Features Needed** (list, create, edit, delete, search, pagination)
- **Additional State** (any special UI states beyond isFetching)

### 2. Generate Files in Order
1. **Type Definition** (`src/types/backend.ts` or new file)
2. **API Functions** (`src/config/api.ts` - add new functions)
3. **Redux Slice** (`src/redux/slice/[module]Slice.ts`)
4. **Store Registration** (`src/redux/store.ts` - add to reducer)
5. **Page Component** (`src/pages/[Module]/[PageName].tsx`)
6. **Child Components** if needed (list, form, detail)

### 3. Code Generation Checklist
- [ ] All async thunks for CRUD operations
- [ ] Proper error handling in fulfilled/rejected cases
- [ ] Pagination support if applicable
- [ ] Loading states management
- [ ] Type safety throughout
- [ ] Hook integration (useAppDispatch, useAppSelector)
- [ ] Proper API parameter passing
- [ ] React Hook Form integration for forms
- [ ] Tailwind + shadcn/ui styling
- [ ] Error messaging (using sonner toast)

## Response Format

When generating a module:
1. List all files to be created/modified
2. Provide complete file contents
3. Include setup instructions (store registration, imports)
4. Show example usage in the page component
5. Explain any non-standard patterns

## Quality Standards
- Follow existing TypeScript patterns
- Maintain type safety
- Use Redux Toolkit best practices
- Consistent with project's UI/UX (Tailwind + shadcn)
- Include error handling
- Support pagination for list views
- Clean, readable code
