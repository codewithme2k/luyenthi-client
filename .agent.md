# Redux Page Module Generator Agent

## Purpose
Generate new page modules with Redux integration following the established patterns in the luyenthi-client project.

---

## Code Patterns Reference

### Redux Store Structure
- **Store**: Uses Redux Toolkit with `configureStore`
- **Location**: `src/redux/store.ts`
- **Slices Location**: `src/redux/slice/`
- **Active Slices**:
  - `accountSlice.ts` - Handles current authenticated session state (login, logout, refresh, profile status).
  - `userSlice.ts` - Manages database user list, creation, updating, and deleting.
  - `categorySlice.ts` - Manages preparation categories (THPT, ĐGNL, etc.).
  - `examSlice.ts` - Manages exams lists, duration, total/pass marks, and settings.
  - `postSlice.ts` - Manages blogs, posts, titles, slugs, and publishing status.
  - `questionSlice.ts` - Manages quiz questions, options, point allocations, and answer choices.
- **Hooks**: Custom typed hooks in `src/redux/hooks.ts` (`useAppDispatch`, `useAppSelector`).

### Slice Pattern (Based on `categorySlice.ts`)
```typescript
import { createAsyncThunk, createSlice } from "@reduxjs/toolkit";
import {
  callFetchCategory,
  callCreateCategory,
  callUpdateCategory,
  callDeleteCategory,
} from "@/config/api";
import type { ICategory } from "@/types/backend";

/* =====================
   Thunks
   ===================== */
export const fetchCategory = createAsyncThunk(
  "category/fetchCategory",
  async ({ query }: { query?: string }) => {
    const res = await callFetchCategory(query);
    return res.data;
  }
);

export const createCategory = createAsyncThunk(
  "category/createCategory",
  async (category: Partial<ICategory>) => {
    const res = await callCreateCategory(category);
    return res.data;
  }
);

export const updateCategory = createAsyncThunk(
  "category/updateCategory",
  async ({ category, id }: { category: Partial<ICategory>; id: string }) => {
    const res = await callUpdateCategory(category, id);
    return res.data;
  }
);

export const deleteCategory = createAsyncThunk(
  "category/deleteCategory",
  async ({ id }: { id: string }) => {
    const res = await callDeleteCategory(id);
    return res.data;
  }
);

/* =====================
   State Interface
   ===================== */
interface IState {
  isFetching: boolean;
  meta: {
    limit: number;
    page: number;
    total: number;
  };
  data: ICategory[];
}

const initialState: IState = {
  isFetching: false,
  meta: {
    limit: 10,
    page: 1,
    total: 0,
  },
  data: [],
};

/* =====================
   Slice Definition
   ===================== */
const categorySlice = createSlice({
  name: "category",
  initialState,
  reducers: {},

  extraReducers: (builder) => {
    builder
      // Fetch
      .addCase(fetchCategory.pending, (state) => {
        state.isFetching = true;
      })
      .addCase(fetchCategory.rejected, (state) => {
        state.isFetching = false;
      })
      .addCase(fetchCategory.fulfilled, (state, action) => {
        state.isFetching = false;
        const payload = action.payload;
        if (payload?.success && payload.data) {
          state.data = payload.data;
          if (payload.meta) {
            state.meta = payload.meta;
          }
        } else {
          state.data = [];
          state.meta = initialState.meta;
        }
      })

      // Create
      .addCase(createCategory.pending, (state) => {
        state.isFetching = true;
      })
      .addCase(createCategory.fulfilled, (state, action) => {
        state.isFetching = false;
        const payload = action.payload;
        if (payload?.success && payload.data) {
          state.data.unshift(payload.data);
        }
      })
      .addCase(createCategory.rejected, (state) => {
        state.isFetching = false;
      })

      // Update
      .addCase(updateCategory.pending, (state) => {
        state.isFetching = true;
      })
      .addCase(updateCategory.fulfilled, (state, action) => {
        state.isFetching = false;
        const payload = action.payload;
        if (payload?.success && payload.data) {
          const index = state.data.findIndex((c) => c.id === payload.data?.id);
          if (index !== -1) {
            state.data[index] = payload.data;
          }
        }
      })
      .addCase(updateCategory.rejected, (state) => {
        state.isFetching = false;
      })

      // Delete
      .addCase(deleteCategory.pending, (state) => {
        state.isFetching = true;
      })
      .addCase(deleteCategory.fulfilled, (state, action) => {
        state.isFetching = false;
        if (action.payload?.success) {
          const deletedId = action.meta.arg.id;
          state.data = state.data.filter((c) => c.id !== deletedId);
        }
      })
      .addCase(deleteCategory.rejected, (state) => {
        state.isFetching = false;
      });
  },
});

export default categorySlice.reducer;
```

### API Function Pattern
- **Location**: `src/config/api.ts`
- **Pattern**: Named exports using customization Axios instance. Returns an Axios promise resolving to responses.
- **Example API Functions**:
  - Auth: `callRegister()`, `callLogin()`, `callFetchAccount()`, `callRefreshToken()`, `callLogout()`
  - User: `callFetchUser()`, `callCreateUser()`, `callUpdateUser()`, `callDeleteUser()`
  - Category: `callFetchCategory()`, `callCreateCategory()`, `callUpdateCategory()`, `callDeleteCategory()`
  - Exam: `callFetchExam()`, `callCreateExam()`, `callUpdateExam()`, `callDeleteExam()`
  - Post: `callFetchPost()`, `callCreatePost()`, `callUpdatePost()`, `callDeletePost()`
  - Question: `callFetchQuestion()`, `callCreateQuestion()`, `callUpdateQuestion()`, `callDeleteQuestion()`

### Page Component Pattern
- **Location**: `src/pages/Dashboard/`
- **Available Dashboard Pages**:
  - `CategoryPage.tsx` - Manage examination and blog categories
  - `ExamPage.tsx` - Manage mock-tests, durations, thresholds
  - `PostPage.tsx` - Manage editorial learning materials
  - `QuestionPage.tsx` - Fully interactive interface for questions and options, supporting multiple styles (SINGLE_CHOICE, MULTIPLE_CHOICE, FILL_BLANK, etc.)
  - `UserPage.tsx` - Manage user permissions, profile attributes, and roles
  - `Dashboard.tsx` - High-level metrics view
- **Form Libraries**: Uses `react-hook-form` paired with `@hookform/resolvers` (Zod schemas).
- **Styling UI**: Styled with Tailwind CSS, utilizing Radix-based premium Shadcn components (Dialog, Buttons, Inputs, Selectors) and Sonner for status messages.

### Type Definitions
- **Location**: `src/types/backend.d.ts` (Ambient Declarations)
- **Key Models**:
  - `IBackendRes<T>` - Wrapper matching backend API responses
  - `IModelPaginate<T>` - Standard paginated data payload
  - `IAccount` & `IGetAccount` - Session user states
  - `IUser` & `ICreateUser` - User schemas
  - `ICategory` - Category schemas
  - `IExam` - Test schemas
  - `IPost` - News & articles schema
  - `IQuestion` & `IAnswerOption` - Question configurations (includes types: `'SINGLE_CHOICE' | 'MULTIPLE_CHOICE' | 'FILL_BLANK' | 'ESSAY' | 'TRUE_FALSE' | 'MATCHING'`)

## Generation Requirements

When asked to create/modify page modules, follow these steps:

### 1. Gather Information
Verify:
- **Module/Resource Name** (e.g., "Post", "Question")
- **Schema properties & types** (matching `src/types/backend.d.ts`)
- **API endpoints & query options** (pagination, sorting, filtering keys)
- **Design Layout** (Modal actions, full-page edit, or sliders)

### 2. Generate Files in Order
1. **Type Definition** (`src/types/backend.d.ts` - append interfaces)
2. **API Functions** (`src/config/api.ts` - export new endpoints)
3. **Redux Slice** (`src/redux/slice/[module]Slice.ts`)
4. **Store Registration** (`src/redux/store.ts` - hook reducer)
5. **Page Component** (`src/pages/Dashboard/[ModuleName]Page.tsx`)

### 3. Code Generation Checklist
- [ ] Implement CRUD thunks in Slice
- [ ] Add active states (fetching indicators, modal states)
- [ ] Hook Redux selectors (`useAppSelector`, `useAppDispatch`)
- [ ] React-Hook-Form configuration with custom validation resolvers
- [ ] Beautiful CSS animations and clean premium layouts
- [ ] Toast reports on successful creations, updates, or error blocks

## Quality Standards
- Strict type-safety: Always type API arguments and slice parameters.
- Standard hooks: Rely on pre-typed store hooks.
- Seamless visuals: Ensure smooth hover transitions, glassmorphic accents, and fully responsive cards.
- Clean code: Strict ESLint compliance, proper component structure, and reusable sub-components.
